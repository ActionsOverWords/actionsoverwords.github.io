## Why Split Docs and Blog?

The core decision was to keep:

- category-based technical notes in `src/content/docs/`
- dated entries in `src/content/blog/`

## What This Changes

This keeps the sidebar focused while still allowing timeline-oriented writing. The docs page that explains the authoring rule lives at [Writing Workflow](../../meta/writing-workflow/).

### Editorial Rule

Evergreen notes belong in the docs section, while dated implementation logs and retrospectives belong in the blog.
