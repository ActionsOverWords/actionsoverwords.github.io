## Docs pages

Add technical reference pages under `src/content/docs/` using category paths such as:

- `backend/node/cache.mdx`
- `frontend/astro/content-workflow.mdx`
- `testing/quality-gates.mdx`

Docs pages appear in the Starlight sidebar and should prioritize evergreen knowledge.

## Blog posts

Add dated posts under `src/content/blog/` with metadata for `date`, `tags`, `draft`, and `lastUpdated`.

Blog posts are better for:

- learning logs
- short retrospectives
- implementation notes tied to a date

## Linking

Prefer direct links between docs and blog content when a reference page has a related dated log, or when a post should point back to stable documentation.
