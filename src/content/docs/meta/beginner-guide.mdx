# 초보자 가이드

Astro, Starlight, MDX, 위키형 문서 사이트가 모두 처음이라면 이 페이지부터 보는 것이 가장 좋습니다.

## 전체 구조 한눈에 보기

이 프로젝트의 글은 크게 두 종류입니다.

- `위키 문서`: `src/content/docs/`
- `블로그 글`: `src/content/blog/`

주제별로 오래 정리할 내용은 위키에 쓰고, 날짜 중심 기록은 블로그에 씁니다.

## 각 도구가 하는 일

### Astro

사이트를 실행하고 정적 페이지로 빌드합니다.

### Starlight

문서형 레이아웃, 사이드바, 목차, 검색을 제공합니다.

### MDX

처음에는 거의 Markdown처럼 써도 됩니다. 제목, 목록, 링크, 코드 블록만 써도 충분합니다.

## 위키 문서 추가하기

예시 파일:

`src/content/docs/backend/redis/intro.mdx`

예시 내용:

```mdx
---
title: Redis 입문
description: Redis를 처음 볼 때 필요한 기초 메모
tags:
  - redis
  - backend
draft: false
lastUpdated: 2026-03-25
---

## Redis란

Redis는 캐시나 빠른 키-값 저장소로 자주 사용된다.
```

이 파일은 `/backend/redis/intro/` 같은 주소로 연결됩니다.

## 블로그 글 추가하기

예시 파일:

`src/content/blog/2026-03-26-learning-redis.mdx`

예시 내용:

```mdx
---
title: Redis 공부 시작
description: Redis를 처음 공부하며 정리한 메모
date: 2026-03-26
tags:
  - redis
  - learning
draft: false
lastUpdated: 2026-03-26
---

오늘은 캐시 무효화와 TTL 차이를 정리했다.
```

이 글은 `/blog/` 목록에 나타납니다.

## 가장 많이 쓰는 명령어

```bash
npm run dev
npm run test
ASTRO_TELEMETRY_DISABLED=1 npm run typecheck
ASTRO_TELEMETRY_DISABLED=1 npm run build
ASTRO_TELEMETRY_DISABLED=1 npm run test:e2e
```

## 초보자용 기준

- 오래 보관할 지식은 `위키`
- 날짜가 중요한 기록은 `블로그`
- 처음에는 복잡한 기능 없이 Markdown 스타일 MDX만 사용
