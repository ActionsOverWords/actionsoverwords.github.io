## MDX란?

**MDX**는 Markdown 문서 안에서 **JSX 컴포넌트**를 사용할 수 있게 해주는 형식입니다.
일반 Markdown처럼 작성하되, 필요한 곳에 React/Astro 컴포넌트를 삽입할 수 있습니다.

```mdx
---
title: 내 문서
description: 문서에 대한 한 줄 설명
tags: [가이드, 예시]
---

import { MyComponent } from '../components/MyComponent.astro';

# 제목

일반 Markdown 문장입니다.

<MyComponent name="Web1 Team" />
```

---

## Frontmatter (문서 메타데이터)

모든 `.mdx` 파일은 맨 위의 `---` 블록(frontmatter)에 메타데이터를 작성합니다.

```yaml
---
title: 문서 제목          # (필수) 페이지 제목 및 사이드바 표시명
description: 한 줄 설명   # (권장) 검색 결과·미리보기에 표시
tags: [태그1, 태그2]      # (선택) /tags/ 페이지에서 필터링 가능
---
```

### 주요 필드

| 필드 | 필수 | 설명 |
|------|:----:|------|
| `title` | ✅ | 페이지 `<title>` 및 사이드바에 표시되는 이름 |
| `description` | 권장 | SEO 메타 설명, 링크 카드 미리보기에 사용 |
| `tags` | ❌ | 문자열 배열. `/tags/` 페이지에서 태그별 문서 탐색 가능 |
| `sidebar.order` | ❌ | 사이드바 정렬 순서 (숫자가 작을수록 위) |
| `sidebar.label` | ❌ | 사이드바에 표시할 이름 (`title`과 다르게 하고 싶을 때) |
| `draft` | ❌ | `true`로 설정하면 프로덕션 빌드에서 제외 |

<Aside type="tip" title="tags 활용">
`tags`를 지정하면 `/tags/` 페이지에서 관련 문서를 한눈에 찾을 수 있습니다.

```yaml
tags: [XRM, API, 가이드]
```
</Aside>

### 사이드바 순서 지정 예시

같은 디렉터리 내 문서 순서를 직접 제어하려면 `sidebar.order`를 사용합니다.

```yaml
---
title: 시작하기
sidebar:
  order: 1
---
```

## 컴포넌트 Import

`.mdx` 파일 안에서 frontmatter 바로 아래에 `import` 구문을 작성합니다.

```js
import { Aside, Card } from '@astrojs/starlight/components';
import MyChart from '../../components/MyChart.astro';
```

<Aside>
`import` 구문은 반드시 frontmatter(`---`) 블록 **아래**, 본문 내용 **위**에 위치해야 합니다.
</Aside>

## 표현식(Expressions)

MDX에서는 `{}` 안에 JavaScript 표현식을 사용할 수 있습니다.

```js
export const version = '2.1.0';

현재 버전: **{version}**

오늘 날짜: {new Date().toLocaleDateString('ko-KR')}
```

## 주의사항

### HTML 태그와의 차이

MDX에서는 JSX 규칙을 따르므로 일반 HTML과 차이가 있습니다.

| HTML | MDX(JSX) |
|------|---------|
| `class="..."` | `className="..."` |
| `<br>` | `<br />` |
| `<img src="">` | `<img src="" />` |

### 빈 줄 구분

컴포넌트와 Markdown 텍스트 사이에는 빈 줄을 두어야 Markdown이 올바르게 파싱됩니다.

```jsx
<Aside>내용</Aside>

여기에 일반 텍스트를 씁니다. (빈 줄 필요)
```

## 파일 확장자

| 확장자 | 설명 |
|--------|------|
| `.md` | 순수 Markdown, 컴포넌트 사용 불가 |
| `.mdx` | MDX, 컴포넌트 import 및 JSX 사용 가능 |

<Aside type="tip">
컴포넌트가 필요 없는 단순 문서는 `.md`를 사용해도 됩니다.
Starlight는 두 형식 모두 지원합니다.
</Aside>
